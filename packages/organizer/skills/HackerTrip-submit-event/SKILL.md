# 把黑客松活动提交到 HackerTrip 平台（submit-event）

帮主办方**零门槛**把活动提交到 HackerTrip：主办方先在小程序「主办方后台」生成 6 位配对码把这次提交**绑定到自己的小程序账号**，再口述活动信息或丢一张海报图，skill 自动整理成标准字段、过一遍准入门槛 + 格式自检，提交到平台草稿箱。提交进的是**绑账号的草稿 + 平台审核台人工审核**，不会直接公开——这既是质量门槛，也保护主办方不被错误信息坑。

> 适用对象：黑客松主办方 / 承办方 / 社区组织者。
> 不适用：选手发作品（用 `HackerTrip-publish-work`）、批量采集第三方赛事入库（那是平台运营侧的 `HackerTrip-hackathon-intake`）。

---

## 一、流程总览

```
小程序「主办方后台」生成配对码+token → 收集活动信息(口述 / 海报图OCR)
   → 过【准入门槛自检(4道GATE) + 格式自检】 → submit-event --pair-code 提交
   → CLI 校验 → 云端再校验 → 进【绑账号草稿】待审核台人工审核
                                          │
              任一层校验不齐全 / 格式错 → 直接拒绝，不进库
              缺必填(name/时间/地点/官网) → 提示补全，不提交
              奖金/日期不确定 → 如实留空，绝不编造
```

> **双层格式校验**贯穿全程：skill 提交前先自检 → CLI 本地再校验 → 云端入库前还会校验一遍。**任一层发现字段不齐或格式不对，提交会被直接拒绝，不进库。**

---

## 二、前置准备

### 1. 安装 CLI
```bash
npx hackertrip --help   # 或 npm i -g hackertrip
```

### 2. 在小程序生成配对码 + token（绑定主办方账号）
引导用户：打开 **HackerTrip 小程序 → 主办方后台 → 生成提交配对码**，会拿到：
- **6 位配对码**（如 `482915`）——一次性、有时效，把这次提交**绑定到主办方本人的小程序账号**。
- **token**——提交鉴权用，对应 CLI 的环境变量 `HACKERTRIP_SYNC_TOKEN`。

### 3. 配置环境变量
```bash
export HACKERTRIP_SYNC_TOKEN="<上一步主办方后台给出的 token>"
```
> 缺失时 CLI 会报错提示。配对码在命令里用 `--pair-code` 传入，赛事会因此绑定到该主办方账号。

---

## 三、第 1 步：收集活动信息

两种输入都支持，目标是凑齐下面这些字段：

| 字段 | 参数 | 说明 |
|---|---|---|
| 活动名称 | `--name` | **必填** |
| 城市 / 地点 | `--city` | **必填**（线上填"线上"） |
| 开始日期 | `--start` | **必填**，`YYYY-MM-DD` |
| 结束日期 | `--end` | **必填**，`YYYY-MM-DD` |
| 官网 / 报名链接 | `--website` | **必填** |
| 形式 | `--mode` | `offline` / `online` / `hybrid` |
| 奖金 | `--prize` | 不确定就留空，别编 |
| 赛道 | `--tracks` | 逗号分隔，如 `AI Agent,Web3` |
| 一句话简介 | `--summary` | 给选手看的活动亮点 |
| 主办方 | `--organizer` | 谁办的 |
| 联系方式 | `--contact` | 报名/咨询入口 |

### 输入方式 A：用户口述
逐项确认，缺必填项就追问。

### 输入方式 B：用户丢海报图
对海报做**视觉 OCR 提取**：读取图片，把名称 / 时间 / 地点 / 官网 / 报名截止 / 奖金 / 赛道 / 主办方 / 联系方式逐一抠出来。
> 海报上没写的字段就当缺失，**不要脑补**。抠完后向用户复述一遍让其确认/补全。

---

## 四、第 2 步：双层校验之第一层 — 本地自检（准入门槛 + 格式）

提交前 skill 本地**必须**先过两组自检，任一不过就提示主办方补全/修正，**不要硬提交**。这是双层校验的第一层；CLI 和云端还会各自再校验一遍。

### A. 准入门槛自检（4 道 GATE，复用 hackathon-intake 标准）

逐条过这 4 道门槛（直接引用 `HackerTrip-hackathon-intake` 的准入标准）：

### 门槛 1：类型 — 必须是"真黑客松"
限时开发 / 驻场 / 马拉松式（N 小时或 N 天连续开发 + 提交可运行作品 + 评审）。
排除：算法评测赛、创业计划/路演赛、训练营/夏令营、作品征集赛、纯 Demo Day、内部员工赛、长期常驻算法赛。

### 门槛 2：时效性 — 必须未过期
报名截止 ≥ 今天，或赛期在未来。**务必核对年份**，别把往届的当成新一届。已结束的活动不提交。

### 门槛 3：真实性 — 信息属实
关键数字（奖金、日期）要是真的；主办方自己提交时尤其要诚实，**假信息审核会被拒**。

### 门槛 4：完整度 — 核心字段齐
**必填**：`name` + 开始/结束时间 + 城市/地点 + 官网（或报名链接）。缺任一就提示补全，不提交。
`prize` / `tracks` / `contact` 是加分项，没有可留空。

### B. 格式自检（提交前必过，CLI/云端会再校验同样规则）

- **日期**：`--start` / `--end` 必须是 `YYYY-MM-DD`；不是这个格式直接修正，别提交。
- **起止先后**：`--start` ≤ `--end`，开始不能晚于结束。
- **URL**：`--website` 必须以 `http://` 或 `https://` 开头，是真实可访问链接。
- **必填齐全**：`--pair-code` + `name` + `start` + `end` + `city` + `website` 一个都不能少。

> 一句话给主办方讲清楚：**奖金、日期不确定就如实留空，不要编造；字段不齐或日期/URL 格式不对，CLI 和云端会直接拒掉、不进库；提交即进绑账号草稿，由平台审核台人工审核，假信息/垃圾会被拒。**

---

## 五、第 3 步：提交（CLI + 云端第二、三层校验）

### 方式 A：命令行直接传字段
```bash
hackertrip submit-event \
  --pair-code 482915 \
  --name "AdventureX 2026 黑客松" \
  --city "杭州" \
  --start 2026-07-25 \
  --end 2026-07-27 \
  --website "https://adventure-x.org" \
  --mode offline \
  --prize "总奖池 50 万元" \
  --tracks "AI Agent,硬件,Web3" \
  --summary "全国最大青年黑客松，48 小时连续开发" \
  --organizer "AdventureX 组委会" \
  --contact "公众号 AdventureX 后台报名"
```
- `--pair-code` **必填**，就是主办方后台那 6 位码，赛事会绑定到该主办方账号。
- CLI 会先在本地做一遍格式校验（日期 `YYYY-MM-DD`、URL `http(s)`、起止先后、必填齐全），**不齐全/格式错当场报错、不发送**；通过后云端入库前还会再校验一遍，云端拒绝也不进库。

### 方式 B：从文件提交（信息多时更清爽）
先整理成 `event.json`：
```json
{
  "name": "AdventureX 2026 黑客松",
  "city": "杭州",
  "start": "2026-07-25",
  "end": "2026-07-27",
  "website": "https://adventure-x.org",
  "mode": "offline",
  "prize": "总奖池 50 万元",
  "tracks": ["AI Agent", "硬件", "Web3"],
  "summary": "全国最大青年黑客松，48 小时连续开发",
  "organizer": "AdventureX 组委会",
  "contact": "公众号 AdventureX 后台报名"
}
```
再提交（配对码仍要在命令行传）：
```bash
hackertrip submit-event --pair-code 482915 --from event.json
```
> `--from` 文件里的字段会作为基底，命令行同名参数可覆盖个别项。`--pair-code` 不能写进文件，必须命令行传，用于绑定主办方账号。

---

## 六、第 4 步：告知结果

提交成功后告诉主办方：**活动已绑定你的小程序账号、进入 HackerTrip 草稿箱，等待平台审核台人工审核，审核通过后才会对选手公开。** 期间若有信息需补充，可用同一配对码重新提交修正。
若 CLI 或云端报「字段不齐 / 格式错误」被拒，按提示补全格式（日期、URL、起止先后、必填项）后重提，**被拒的不会进库**。

## 七、自检清单

- [ ] CLI 可用、`HACKERTRIP_SYNC_TOKEN` 已配置
- [ ] 已在小程序「主办方后台」生成 6 位配对码（绑账号）
- [ ] 信息已收齐（口述确认 / 海报 OCR 后复述确认）
- [ ] 过门槛 1：是真黑客松（非评测/创业赛/训练营/征集/路演/内部赛）
- [ ] 过门槛 2：未过期、年份已核对
- [ ] 过门槛 3：奖金/日期属实，不确定的已留空
- [ ] 过门槛 4：name + 时间 + 地点 + 官网 四项必填齐全
- [ ] 格式自检：日期 `YYYY-MM-DD`、URL `http(s)`、start ≤ end
- [ ] `submit-event --pair-code` 提交成功（CLI/云端两层校验均通过）
- [ ] 已告知主办方「已绑账号进草稿、待审核台人工审核」
